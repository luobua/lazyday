# 2026-04-25 工作记忆

## 项目概述
Lazyday 多租户 API 开放平台（AI 大脑架构），腾讯内部 CodeBuddy AI IDE 相关项目。

- Backend: Spring Boot 4.0.6 + WebFlux + R2DBC + PostgreSQL + PGvector

- Frontend: Next.js 15 + TypeScript + Ant Design 5.x（Monorepo）
- AI: Spring AI 2.0.0-M4 + DashScope(Qwen)

## 核心概念澄清：中枢 vs 大脑
用户明确架构概念：**Frontend + Backend = 中枢（管理/监控/调度）**，**Edge = 大脑（AI 执行单元）**。
5 份文档全部更新至 v0.4，体现此分工：
- **中枢（Frontend + Backend）**：管理界面 + 核心服务，负责 AI 能力的管理、配置、监控、调度，不直接执行 AI 推理
- **大脑（Edge）**：AI 执行单元，接收中枢下发的配置后组装 BrainContext，执行 RAG/Agent/Workflow/TTS/ASR，不存储配置

关键更新：
- `architecture-overview.md`：架构图重绘，中枢/大脑分层标注；§3 模块说明重新划分（Frontend=中枢界面，Backend=中枢服务，Edge=大脑执行）

## 迭代计划（2026-04-26）

基于 5 份架构文档的完整功能清单，划分 5 个迭代阶段：

### Phase 1（8-10 周）：基础开放平台
**交付目标**：开发者可注册租户、创建 AppKey、通过 Open API 验证身份并调用基础管理接口

Backend：
- Flyway V1 迁移脚本（t_tenant / t_user / t_app_key）
- 租户域（TenantAggregation + TenantEntity + TenantRepository）
- AppKey 域（AppKeyAggregation + AppKeyEntity + AppKeyRepository + SecretKey AES 加密）
- 鉴权接口（JWT 登录/注册/Refresh Token）
- Portal API（租户注册 + AppKey CRUD + SecretKey 轮换宽限期）
- Admin API（租户列表 + 暂停/恢复）
- 内部 API（/internal/appkey/validate 供 Edge 调用）

Edge：
- 项目骨架（Spring Cloud Gateway 4.x + WebFlux）
- TenantContextFilter + AuthGlobalFilter（HMAC 验签 + AppKey Redis 缓存）
- 路由配置（/api/open/v1/* → forward；/api/portal/v1/* + /api/admin/v1/* → HTTP 转发 Backend）
- BackendClient（WebClient）
- AppWebFilter（CORS）

Frontend：
- Next.js Monorepo（Portal + Admin 双应用 + Turborepo）
- packages/api-client（Axios + Portal API 接口）
- Portal 登录/注册页面（SSR + TanStack Query）
- Portal AppKey 管理页面（列表 + 创建 + 禁用/启用 + SecretKey 轮换）
- Middleware 路由守卫（JWT Cookie）

**验收标准**：注册 → 创建 AppKey → HMAC 签名调用 Open API → 鉴权通过

### Phase 2（6-8 周）：配额与可观测性
**交付目标**：平台具备流量控制和透明可观测能力

- Flyway V2 迁移脚本（t_quota_plan / t_tenant_quota）
- Backend 配额域（QuotaPlan CRUD + 租户绑定 + 套餐动态修改实时生效）
- Backend 调用日志域（t_call_log 按月分区 + 统计聚合 API）
- Backend WebSocket 服务端（Edge 注册 + 心跳 + 批量日志接收）
- Edge RateLimitGlobalFilter（QPS 令牌桶 + 日/月配额 Redis 计数器）
- Edge CallLogGlobalFilter（Sinks.Many buffer 批量推送 WebSocket）
- Portal 日志查询页面（时间筛选 + 分页）
- Portal 统计看板（ECharts 折线图 + 饼图）

**验收标准**：QPS 超限返回 429；日志 5 秒内可查；统计图表数据准确

### Phase 3（6-8 周）：Webhook 与运营后台
**交付目标**：平台具备事件驱动能力和完整运营管理界面

- Flyway V3 迁移脚本（t_webhook_config / t_webhook_event）
- Backend Webhook 域（订阅配置 + DomainEventPublisher + 指数退避重试）
- Backend Webhook Portal API（CRUD + 测试推送）
- 邮件服务（Spring Mail + 阿里云邮件推送：注册验证 + 配额告警）
- Portal Webhook 管理页面（创建/编辑/测试）
- Admin Console（Next.js Monorepo admin 应用）
- Admin 租户管理（列表 + 详情 + 暂停/恢复/改套餐）
- Admin 套餐管理（CRUD + 绑定租户数）
- Admin 系统概览大盘

**验收标准**：Webhook 事件触发后 10 秒内收到推送；Admin 可管理所有租户

### Phase 4（10-12 周）：AI 大脑
**交付目标**：租户可配置 AI 大脑，Edge 大脑组装执行端到端 AI 请求

- Flyway V4 迁移脚本（t_rag_config / t_rag_document / t_agent_config / t_workflow_config / t_workflow_node / t_tenant_brain_config）
- Backend RAG 配置域（知识库 CRUD + 文档上传 + PGvector 向量化 + 状态机）
- Backend Agent 配置域（角色 + 提示词 + 模型 + 工具 + TTS/ASR JSON 配置）
- Backend Workflow 配置域（流程 CRUD + 节点类型 + 乐观锁版本）
- Backend 配置下发域（TenantBrainConfig JSON 快照 + WebSocket 推送）
- Edge 大脑组装（BrainAssemblyService：ToolRegistry + L1 Caffeine/L2 Redis + stale-while-revalidate）
- Edge 大脑执行器（RAG 检索器 + AIAgent + WorkflowEngine + DashScopeTtsClient + DashScopeAsrClient）
- Edge Open API（/ai/chat + /ai/tts + /ai/asr + /ai/workflow + /ai/agent）
- Frontend RAG 管理页面（文档上传 + 向量化状态 + 检索测试）
- Frontend Agent 配置页面（Monaco Editor + TTS/ASR + 工具多选）
- Frontend Workflow 编排画布（ReactFlow + 版本发布）
- Admin 大脑配置下发状态监控

**验收标准**：配置 → 下发 → Edge 执行 → 响应，完整链路通

### Phase 5（4-6 周）：完善与优化
**交付目标**：平台达到生产就绪状态

- 邮件服务完善（注册验证 + 配额告警 + SecretKey 轮换提醒）
- Workflow 高级节点（循环/并行/子流程）
- Agent 多模态（图片输入 + 语音流式输入）
- 性能压测（Edge 鉴权 P99 ≤ 5ms；非 AI P99 ≤ 200ms；AI chat TTFB ≤ 2s）
- Prometheus + Grafana 接入

## 关键设计决策
1. ID 生成策略：业务表 BIGINT 自增序列，调用日志表雪花算法
2. 配置粒度：t_tenant_brain_config 以租户维度整合（JSON 快照），当前规模足够
3. 会话存储：Redis L2 + Caffeine L1，Edge 不持久化
4. WebSocket 方向：Edge 主动连接 Backend（拉模式）
5. SpEL 安全：SimpleEvaluationContext + 白名单校验

### Frontend 框架搭建完成（2026-04-26）

基于 frontend-architecture.md 搭建了完整的 Next.js Monorepo 前端框架，Portal + Admin 双应用均构建通过并可在开发模式下运行。

**技术栈确认**：
- Next.js 15.5.15 + TypeScript 5.9.3 + Ant Design 5.x
- pnpm 10.33.2 + Turborepo 2.9.6
- Zustand 5.x（客户端状态）+ TanStack Query 5.x（服务端状态）
- Node v24.1.0（系统已有，通过 nvm 管理）

**项目结构**：
```
frontend/
├── apps/portal/          # 开发者控制台（端口 3001, basePath: /portal）
│   ├── app/(auth)/       # 登录/注册/找回密码
│   ├── app/(dashboard)/  # 控制台（概览/AppKey/日志/Webhook/RAG/Agent/Workflow/设置）
│   ├── store/auth.ts     # Zustand 认证状态
│   ├── lib/query-provider.tsx  # TanStack Query Provider
│   └── middleware.ts     # JWT Cookie 路由守卫
├── apps/admin/           # 管理后台（端口 3002, basePath: /admin）
│   ├── app/(auth)/       # Admin 登录
│   ├── app/(dashboard)/  # 后台（概览/租户/套餐/日志/RAG/Agent/Workflow/大脑配置/设置）
│   ├── store/auth.ts
│   ├── lib/query-provider.tsx
│   └── middleware.ts
├── packages/
│   ├── types/            # 共享 TypeScript 类型（API 响应/分页/用户/租户/AppKey/RAG/Agent/Workflow 等）
│   ├── utils/            # 共享工具函数（formatTimestamp/copyToClipboard/maskAppKey 等）
│   ├── api-client/       # Axios 请求层（portalClient + adminClient + Portal API + Admin API 全量定义）
│   └── ui/               # 共享 UI 组件（LazydayProvider + PageHeader）
├── pnpm-workspace.yaml
├── turbo.json
└── tsconfig.json         # 根配置含路径别名 @lazyday/*
```

**踩坑记录**：
- `@ant-design/icons` 没有 `WebhookOutlined`，改用 `SendOutlined`
- `@lazyday/ui` 不应依赖 `next/link`（shared package 不属于特定 Next.js app），Breadcrumb 改为纯文本展示
- Next.js 15 + pnpm workspace 下需要 `transpilePackages` 声明 workspace 包

**启动命令**：
```bash
pnpm dev                # 同时启动 Portal(3001) + Admin(3002)
pnpm --filter @lazyday/portal dev   # 单独启动 Portal
pnpm --filter @lazyday/admin dev    # 单独启动 Admin
```
