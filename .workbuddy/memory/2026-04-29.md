## Admin 静态资源 404 修复
- 根因：`next.config.ts` 中 `assetPrefix: '/admin/'` 在 dev 模式下会让浏览器请求 `/admin/_next/...`，但 dev server 实际挂载在 `/_next/...`
- 修复：`assetPrefix` 改为仅 `production` 环境启用，dev 设为 `undefined`
- 需重启 dev server 才能生效（next.config 变更不热更新）

## Portal 三接口 500 修复
- `/quota`、`/logs`、`/logs/stats` 返回 500，重启后端后 `/quota` 恢复正常
- `/logs` 和 `/logs/stats` 仍报 40001（INVALID_PARAMETER），原因是：
  1. **前端 Query 字段名用 snake_case（start_time/end_time/app_key/status_code），后端 @RequestParam 期望 camelCase（startTime/endTime/appKey/statusCode）**
  2. **overview 页面没传 startTime/endTime，后端要求必填**
- 修复：
  - `CallLogQuery` 类型字段名全部改为 camelCase
  - `use-logs.ts`、`portal.ts` 的 Pick 类型同步更新
  - `logs/page.tsx` 的 Form field 名和 query 构造同步更新
  - `overview/page.tsx` 添加默认时间范围（最近 7 天）
- 后端 @RequestParam name 总结：appKey, statusCode, statusCodeGroup, startTime, endTime, path, granularity

## Admin 403 Forbidden 修复（plans 接口）
- 根因：`CookieUtils.java` 所有 cookie 硬编码 `secure(true)`，浏览器在 localhost HTTP 下不发送 Secure cookie
- 修复：加 `isSecureEnabled()` 方法，仅 `SPRING_PROFILES_ACTIVE` 含 prod/staging 时启用 Secure
- 附带修复：CSRF token 路径写死 Portal，adminClient 应使用 `/api/admin/v1/auth/csrf-token`
- `CookieUtilsTest` 同步更新：dev 环境 expect `isSecure() == false`
