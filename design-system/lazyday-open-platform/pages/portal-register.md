## 页面定位

开发者注册入口。注册 = 创建用户 + 创建租户，一步完成。字段尽量少，降低注册摩擦。

- 与登录页保持**完全一致**的卡片布局、尺寸、背景色
- 表单字段 ≤ 5 个（用户名、邮箱、密码、确认密码、租户名称）

## 表单字段

| 字段 | 类型 | 校验规则 | 校验时机 |
|------|------|---------|---------|
| 用户名 | `<Input>` | 必填, 3-32 字符, 字母数字下划线 | onBlur |
| 邮箱 | `<Input>` | 必填, 邮箱格式 | onBlur |
| 密码 | `<Input.Password>` | 必填, ≥ 8 字符, 含大小写+数字 | onBlur |
| 确认密码 | `<Input.Password>` | 必填, 与密码一致 | onBlur |
| 租户名称 | `<Input>` | 必填, 2-64 字符 | onBlur |

### 密码强度提示
- 密码字段下方展示强度指示器（弱/中/强）
- 使用三段色条：红 → 橙 → 绿
- 规则提示文字: "至少 8 位，包含大小写字母和数字"
- color: `#94A3B8`，font-size: 12px

## 交互流程

```
用户填写 → 点击注册
  ├── 前端校验全部通过
  ├── Button loading=true
  ├── POST /api/portal/v1/auth/register
  │     body: { username, email, password, tenant_name }
  ├── 成功 → message.success('注册成功')
  │         → 自动登录（Backend 返回 JWT Cookie）
  │         → router.push('/overview')
  └── 失败
       ├── 409 用户名已存在 → 内联错误 "用户名已被使用"，聚焦用户名字段
       ├── 409 邮箱已存在 → 内联错误 "邮箱已被注册"
       └── 其他 → message.error(error.message)
```

## 错误状态

| 场景 | 展示方式 |
|------|---------|
| 字段为空 | 内联红色提示 |
| 密码不一致 | 确认密码字段内联 "两次密码不一致" |
| 密码强度不够 | 内联 "密码强度不足" + 强度条为红色 |
| 用户名/邮箱冲突 | 内联错误在对应字段下 + 聚焦该字段 |
| 网络错误 | `message.error('网络异常，请稍后重试')` |

## 可访问性要点

- Tab 顺序: 用户名 → 邮箱 → 密码 → 确认密码 → 租户名称 → 注册 → 登录链接
- 密码强度变化时 `aria-live="polite"` 播报
- 所有字段有显式 `<label>`
